# Handoffs

This folder stores active handoff notes for unfinished, blocked, long-running, or fresh-chat continuation work.

Handoffs are bridges between Codex chats. They are not archives and not session transcripts.

## When To Create A Handoff

Create or update a handoff when:

- a Codex thread is long, compacted, token-expensive, or awkward to continue,
- work is unfinished and likely to resume later,
- work is blocked and the blocker/next action must not be lost,
- a project needs a fresh-chat starter.

Do not create a handoff when a normal session note is enough.

## Naming

Use:

```text
YYYY-MM-DD-<project-or-topic>-handoff.md
```

If an active handoff already exists for the same project/topic, update it instead of creating a duplicate.
