# Handoff

Status: active

Use this when work is unfinished, blocked, or the current chat has become long, compacted, token-expensive, or awkward to continue.

Store active handoffs in:

```text
handoffs/YYYY-MM-DD-<project-or-topic>-handoff.md
```

## Current Objective

What is the active goal?

## Why This Matters

Why is this work being done?

## Completed

What has already been finished?

For generated artifacts, label status clearly:

- canonical
- delivery
- diagnostic
- temporary
- superseded

## In Progress

What is underway?

## Blockers Or Risks

What is unclear, risky, broken, or waiting?

## Exact Next Steps

List immediate next actions in order.

## What To Re-Read First

List the minimum notes/files/artifacts needed before acting.

Mark whether each item is `active`, `stable`, `candidate`, `archived`, or `superseded` when freshness matters.

Use clickable absolute markdown links for local artifacts in user-facing handoff starters when possible.

Use source labels when provenance matters:

- `User-stated`
- `Observed in repo`
- `Tool result`
- `Agent inference`
- `External source`

## Fresh Chat Starter

Write a compact paragraph the user can paste into a new Codex chat.

## References

Include file paths, session notes, commits, outputs, or other materials.
